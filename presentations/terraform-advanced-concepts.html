ğŸ“ Terraform Variables & Outputs

ğŸ”¤ Input Variables

Like function parameters - customize configurations without changing source code

Cookie Recipe Analogy: Flour, sugar, butter are your inputs

ğŸ“¤ Output Variables

Return useful values after Terraform applies - like product labels showing what you made

Result: Public IPs, DNS names, ARNs

Input Variable Example

variable "region" { description = "AWS region to deploy resources" type = string default = "us-east-1" } # Usage in other files: provider "aws" { region = var.region }

Output Variable Example

output "ec2_public_ip" { description = "Public IP of EC2 instance" value = aws_instance.web.public_ip } # Terminal Output: # ec2_public_ip = "54.123.45.67"

Ways to Pass Variables

# Via CLI terraform apply -var="instance_type=t3.micro" # Via .tfvars file terraform apply -var-file="prod.tfvars" # Via Environment Variables export TF_VAR_instance_type="t3.micro"

âš™ï¸ TFVars & Variable Overriding

ğŸ¯ Variable Precedence Order (Highest to Lowest)

  1. CLI -var flag: -var="region=us-west-1"
  2. CLI -var-file: -var-file="prod.tfvars"
  3. Environment variables: TF_VAR_region=us-east-2
  4. *.auto.tfvars files: dev.auto.tfvars
  5. Default values: default = "us-east-1"

Example .tfvars File

region = "us-west-2" instance_type = "t3.medium" tags = { Environment = "production" Owner = "DevOps Team" }

Usage Examples

# Manual usage terraform apply -var-file="prod.tfvars" # Multiple files terraform apply -var-file="prod.tfvars" -var-file="secrets.tfvars" # Auto-loading (no CLI needed) # Files named: terraform.tfvars or *.auto.tfvars
