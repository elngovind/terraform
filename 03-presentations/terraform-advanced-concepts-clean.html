Quick Recap - Previous Lecture

What is Infrastructure in DevOps?

Infrastructure includes everything that supports application deployment—servers, storage, networking, IAM, monitoring, etc.

Why IaC Matters

Manual setup is slow, error-prone, and inconsistent.

IaC solves this by turning infrastructure into repeatable, testable, and version-controlled code.

Types of IaC Approaches

Declarative (e.g., Terraform): You define what you want.

Imperative (e.g., Ansible): You define how to do it step-by-step.

Terraform is a declarative tool — it figures out the how for you.

Why Choose Terraform

Supports multi-cloud (AWS, Azure, GCP)

Written in HCL (HashiCorp Configuration Language)

Maintains state to manage infrastructure over time

Has rich support for modules, providers, and collaboration via Git

Terraform Architecture

Terraform Core: The brain that parses .tf files and builds a plan.

Providers: Plugins that know how to talk to platforms like AWS, Azure, etc.

State File: Tracks current state of your infrastructure.

Configuration Files: Where you define resources (main.tf, variables.tf, etc.)

Providers & Plugins

Providers are the most common plugins (e.g., aws, google, kubernetes).

Downloaded via terraform init

Plugins enable Terraform to talk to external systems (via APIs)

HCL Basics - Blocks, Arguments, Expressions

HCL Structure

Blocks: Containers for configuration | Arguments: Key-value pairs | Expressions: Computed values

1. Blocks

resource "aws_instance" "web" { ami = "ami-0c55b159cbfafe1f0" instance_type = "t2.micro" } # Block Structure: # block_type "label1" "label2" { # argument = value # }

2. Arguments

instance_type = "t2.micro" # ↑ key ↑ value # Arguments are key-value pairs # Order-independent # Values can be strings, numbers, booleans, lists, maps

3. Expressions

name = "web-${var.environment}" # ↑ String interpolation expression tags = { Name = "web-${upper(var.environment)}" } # ↑ Function call expression

Terraform Variables & Outputs

Input Variables

Act like function parameters - customize configurations without changing source code

Enable reusable, portable, and readable infrastructure code

Output Variables

Return useful values after Terraform applies configuration

Common outputs: Public IPs, DNS names, ARNs, resource IDs

Input Variable Example

variable "region" { description = "AWS region to deploy resources" type = string default = "us-east-1" } # Usage in other files: provider "aws" { region = var.region }

Output Variable Example

output "ec2_public_ip" { description = "Public IP of EC2 instance" value = aws_instance.web.public_ip } # Terminal Output: # ec2_public_ip = "54.123.45.67"

Ways to Pass Variables

# Via CLI terraform apply -var="instance_type=t3.micro" # Via .tfvars file terraform apply -var-file="prod.tfvars" # Via Environment Variables export TF_VAR_instance_type="t3.micro"

TFVars & Variable Overriding

Variable Precedence Order (Highest to Lowest)

  1. CLI -var flag: -var="region=us-west-1"
  2. CLI -var-file: -var-file="prod.tfvars"
  3. Environment variables: TF_VAR_region=us-east-2
  4. *.auto.tfvars files: dev.auto.tfvars
  5. Default values: default = "us-east-1"

Example .tfvars File

region = "us-west-2" instance_type = "t3.medium" tags = { Environment = "production" Owner = "DevOps Team" }

Usage Examples

# Manual usage terraform apply -var-file="prod.tfvars" # Multiple files terraform apply -var-file="prod.tfvars" -var-file="secrets.tfvars" # Auto-loading (no CLI needed) # Files named: terraform.tfvars or *.auto.tfvars

Hands-On: Parametrized EC2 + VPC Demo

Project Structure

terraform-variables-demo/ ├── main.tf # All resource definitions ├── variables.tf # Input variable declarations ├── outputs.tf # Output variables └── dev.tfvars # Environment-specific values

variables.tf

variable "region" { type = string description = "AWS region where infrastructure will be deployed" } variable "vpc_cidr" { type = string default = "10.0.0.0/16" description = "CIDR block for the VPC" } variable "public_subnet_cidrs" { type = list(string) default = ["10.0.1.0/24", "10.0.2.0/24"] description = "CIDRs for public subnets" } variable "instances" { type = list(object({ name = string })) default = [ { name = "web1" }, { name = "web2" } ] }

main.tf - Key Resources

# VPC resource "aws_vpc" "main" { cidr_block = var.vpc_cidr tags = merge(var.tags, { Name = "main-vpc" }) } # Subnets with for_each loop resource "aws_subnet" "public" { for_each = toset(var.public_subnet_cidrs) vpc_id = aws_vpc.main.id cidr_block = each.value availability_zone = "${var.region}a" map_public_ip_on_launch = true } # EC2 instances with for_each resource "aws_instance" "web" { for_each = { for i, inst in var.instances : inst.name => inst } ami = var.ami_id instance_type = var.instance_type subnet_id = values(aws_subnet.public)[0].id }

Execution Commands

terraform init terraform plan -var-file="dev.tfvars" terraform apply -var-file="dev.tfvars" terraform destroy -var-file="dev.tfvars"
